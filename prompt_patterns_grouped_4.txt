15. 메타언어 생성 패턴 : AI와 대화하는 나만의 비밀 코드
================================================================================
AI에게 특정 명령어(/명령어)에 대한 행동 지침을 미리 학습시켜,
긴 설명 없이도 원하는 결과물을 즉시 얻어내는 고도의 프롬프트 기법.

💡 핵심 원리
- 매번 긴 조건을 반복 입력할 필요 없이 단축 명령어로 즉시 실행
- 명령어는 대화 시작 시 한 번만 학습시키면 해당 대화 내내 유효
- 명령어를 조합(Multi-Command)하면 복합 결과물도 한 번에 생성 가능

흐름 : [명령어 사전 학습] → [단축 명령어 입력] → [정밀 결과물 즉시 출력]

────────────────────────────────────────────────────────────────────────────────
▶ 패턴 A : 기본 명령어 정의 (단일 명령어)
   자주 쓰는 출력 형식을 명령어 하나로 고정.
────────────────────────────────────────────────────────────────────────────────

【 만능 템플릿 】

/[{명령어}] = [{출력 형식}], [{조건1}], [{조건2}], [{조건3}]

(이후 /[{명령어}] 만 입력하면 위 조건을 자동 적용하여 출력)

- - - - - - - - - - - - - - - - - -
【 예시 — 무역 실무용 기본 명령어 세트 】

아래 명령어를 학습하세요. 이후 해당 명령어 입력 시 정의된 방식으로만 출력하세요.

/요약 = 핵심만 3줄, 전문용어 제거, 무역 실무자 기준, [결론 → 이유] 순서로 작성

/코드 = Python 작성, 주석 한국어, 예외처리 포함, 실행 가능한 완성 코드만 출력 (설명 생략)

/비교 = 2~4개 대안을 표로 정리.
        항목: [장점 | 단점 | 비용 | 난이도 | 추천상황]
        마지막 행에 종합 추천 추가

/아이디어 = 창의적이고 실현 가능한 대안 5가지 제시, 각 아이디어별 예상 효과 및 리스크 포함

/검토 = 작성된 내용의 논리적 오류·누락된 점을 비판적으로 분석하고 개선 방향 제시

/가이드 = 초보자가 따라 할 수 있는 Step-by-Step 절차 구성, 단계별 주의사항 포함

- - - - - - - - - - - - - - - - - -
【 AI 예상 응답 (학습 후 /비교 사용 시) 】

사용자 입력 : /비교 엑셀 수작업 vs 파이썬 자동화 vs 노코드 툴

| 항목     | 엑셀 수작업       | 파이썬 자동화    | 노코드 툴         |
|----------|------------------|-----------------|------------------|
| 장점     | 즉시 시작 가능   | 완전 자동화      | 코딩 불필요       |
| 단점     | 오류·반복 작업   | 학습 시간 필요   | 기능 제한         |
| 비용     | 무료             | 무료             | 월 5~20만원       |
| 난이도   | 낮음             | 높음             | 중간              |
| 추천상황 | 소량·비정기      | 대량·정기 반복   | 비개발자·중간규모  |
| 종합추천 | -                | -                | ✅ 노코드 툴 우선 추천 (개발 인력 없는 경우) |

────────────────────────────────────────────────────────────────────────────────
▶ 패턴 B : 변수 기반 만능 명령어 (가변 결과물 생성)
   [대상]과 [옵션]을 바꿔가며 동일 형식의 결과물을 반복 생성.
   구조 : /명령어:[대상]:[옵션]
────────────────────────────────────────────────────────────────────────────────

【 만능 템플릿 】

/[{명령어}]:[{대상}]:[{옵션}] 라고 요청하면,
[{대상}]을 [{옵션}]에 맞춰 처리하고 [{추가 지시사항}]을 수행하라.

- - - - - - - - - - - - - - - - - -
【 예시 — 출판사 편집자 모드 】

[시스템 설정]
[독자] = 초등학생
[역할] = 출판사 편집자 (교양 도서 교정 중)

[명령어 학습]
/유:[단어]:[개수] = [단어]에 대해 [독자]가 이해할 수 있는 유의어 [개수] 제공
                   + 가장 추천하는 유의어 선정

/반:[단어]:[개수] = [단어]에 대해 [독자]가 이해할 수 있는 반의어 [개수] 제공
                   + 가장 추천하는 반의어 선정

/유반:[단어]:[개수] = 유의어와 반의어 검색을 동시에 수행

- - - - - - - - - - - - - - - - - -
【 AI 예상 응답 】

사용자 입력 : /유반:신중하다:2

유의어 (초등학생 수준)
  1. 조심스럽다
  2. 꼼꼼하다
  추천 : '조심스럽다' (일상에서 가장 많이 쓰여요!)

반의어 (초등학생 수준)
  1. 덜렁대다
  2. 성급하다
  추천 : '덜렁대다' (반대 상황을 설명하기 가장 쉬워요!)

────────────────────────────────────────────────────────────────────────────────
▶ 패턴 C : 복합 명령 & 운영 팁 (Multi-Command)
   명령어를 조합하면 한 번에 복합 결과물 생성 가능.
────────────────────────────────────────────────────────────────────────────────

【 만능 템플릿 】

/[{명령어1}] /[{명령어2}] [{처리할 내용}]

- - - - - - - - - - - - - - - - - -
【 예시 】

/요약 /비교  FTA와 MFN 관세율의 차이
→ 3줄 요약 + 비교표를 동시에 생성

/검토 /가이드  아래 수출 절차 문서를 검토하고 개선된 가이드로 재작성해줘
→ 논리 오류 분석 후 Step-by-Step 절차로 재구성

- - - - - - - - - - - - - - - - - -
【 상태 초기화 명령어 】

/리셋 = 이전의 모든 맥락을 삭제하고 메타언어 규칙만 남긴 초기 상태로 복구

(대화가 길어져 맥락이 꼬일 때 사용)

────────────────────────────────────────────────────────────────────────────────

🔑 핵심 포인트
- 명령어는 대화 시작 시 한 번에 몰아서 학습시킬 것
- [독자], [역할] 같은 변수를 먼저 설정하면 모든 명령어에 일관성이 생김
- 명령어가 많아질수록 /리셋으로 초기화 옵션을 함께 설정해둘 것
- 변수 기반 구조(/명령어:[대상]:[옵션])는 반복 작업에 특히 강력함

⚠️ 자주 하는 실수
- 명령어 정의 없이 /명령어만 입력 → AI가 임의로 해석
- 옵션을 너무 많이 넣음 → 핵심 조건 2~3개로 압축할 것
- 대화가 길어진 후 명령어 재학습 없이 사용 → /리셋 후 재학습 필요

19. 무한 생성 패턴 : 무한한 맞춤형 출력 생성하기
================================================================================
고정 프레임(변하지 않는 구조)과 변수(바꿀 수 있는 값)를 분리하여
변수 조합만 바꾸면 동일한 형식의 결과물을 무한히 생성하는 패턴.
더미 텍스트 대량 생성, 이메일 템플릿, 상품 설명, 공문서 초안 등에 특히 강력.

💡 핵심 원리
- 고정 프레임 : 매번 동일하게 유지되는 구조·형식·규칙
- 변수         : 상황마다 바꿔 넣는 값 (수신자·목적·톤 등)
- 생성 규칙    : 분량·단락 수·언어·포함 항목 등 출력 조건
- 변수 조합 수 = 각 변수의 선택지 수를 곱한 값
  (예: 수신자 4 × 목적 4 × 톤 3 = 최대 48가지 조합 자동 생성 가능)

흐름 : [고정 프레임 정의] → [변수 목록 정의] → [생성 규칙 지정] → [조합 입력 → 즉시 출력]

────────────────────────────────────────────────────────────────────────────────
【 만능 템플릿 】

고정 프레임 :
  "[{변하지 않는 출력 구조 설명}]"

변수 :
  {변수1} : [{선택지A}] / [{선택지B}] / [{선택지C}] / [{선택지D}]
  {변수2} : [{선택지A}] / [{선택지B}] / [{선택지C}] / [{선택지D}]
  {변수3} : [{선택지A}] / [{선택지B}] / [{선택지C}]

생성 규칙 :
  - [{분량 조건}]
  - [{구조 조건}]
  - [{언어·형식 조건}]
  - [{기타 제약}]

▶ 사용법 : 위 변수에서 각각 하나씩 골라 아래 형식으로 입력
  → {변수1} : [선택값] / {변수2} : [선택값] / {변수3} : [선택값]

- - - - - - - - - - - - - - - - - -
【 예시 A — 무역 이메일 생성기 】

고정 프레임 :
  "[{수신자 유형}]에게 [{목적}]을 전달하는 영문 비즈니스 이메일"

변수 :
  {수신자} : 신규 바이어 / 기존 거래처 / 포워더 / 관세사
  {목적}   : 첫 연락 / 견적 요청 / 납기 연장 / 클레임 대응
  {톤}     : 공식적 / 친근한 / 긴급한

생성 규칙 :
  - 제목(Subject Line) 포함
  - 본문 3단락 구조 (인사 및 목적 → 핵심 내용 → 다음 단계 요청)
  - 총 150단어 이내
  - 영문 작성

▶ 사용법 : 변수에서 각각 하나씩 골라 입력
  예) {수신자} : 신규 바이어 / {목적} : 첫 연락 / {톤} : 공식적

- - - - - - - - - - - - - - - - - -

【 AI 예상 응답 — 조합 3 : 포워더 / 견적 요청 / 공식적 】

[Name] | [Title] | [Company]

【 예시 B — 상품 설명 대량 생성기 (쇼핑몰 더미 텍스트) 】

🔑 핵심 포인트

⚠️ 자주 하는 실수
- 변수를 너무 많이 만들어 프레임이 복잡해지는 경우
  → 핵심 변수 3개 이내로 시작하고 필요 시 추가
- 생성 규칙 없이 변수만 정의하는 경우
  → 분량·구조·언어를 반드시 함께 명시할 것
- 조합을 하나씩만 요청하는 경우
  → 필요한 조합을 한 번에 묶어 요청하면 시간 절약
