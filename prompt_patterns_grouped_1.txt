13. 질문 개선 패턴 : 질문을 질문하기
================================================================================
AI에게 사용자의 질문 자체를 먼저 분석하고 더 나은 질문으로 업그레이드하게 하는 패턴.
모호하거나 전제가 잘못된 질문을 먼저 교정함으로써 더 정확하고 유용한 답변을 얻음.

💡 핵심 원리
- 질문의 품질이 답변의 품질을 결정함
- 잘못된 전제를 가진 채 답을 받으면 오히려 해가 됨
- AI에게 "먼저 질문을 분석하라"고 지시하면 맥락 오류를 사전에 차단

질문 : FTA가 적용되면 관세를 안 내도 되는 건가요?

- - - - - - - - - - - - - - - - - -
【 AI 예상 응답 】

[전제 교정]
이 질문에는 두 가지 잘못된 전제가 있습니다.
① "FTA 적용 = 관세 면제"가 아닙니다.
   FTA는 관세를 0%로 줄여주는 경우도 있지만,
   협정마다 세율이 다르며 단계적으로 인하되는 품목도 많습니다.
② FTA 혜택을 받으려면 원산지 증명서 발급 등 별도 절차가 필요합니다.
   이를 이행하지 않으면 일반 세율(MFN)이 적용됩니다.

[교정된 질문]
"한-미 FTA 적용 시, 특정 품목(예: HS Code 6109.10)의 관세율이
 얼마나 인하되며, 혜택을 받기 위해 수출자가 준비해야 할
 원산지 증명 절차는 무엇인가요?"

[답변]
한-미 FTA 기준, 대부분의 섬유류는 협정 발효 즉시 0% 또는 단계적 철폐...

📋 질문 보내기 전 셀프 체크리스트

  □ 대상이 명확한가?
    나쁜 예 : "무역에서 AI 활용법은?"
    좋은 예 : "수출 서류 작성 담당 실무자가 Claude를 활용하는 방법은?"

  □ 범위가 좁혀졌는가?
    나쁜 예 : "수출 자동화 방법은?"
    좋은 예 : "엑셀 기반으로 인보이스 자동 생성하는 방법은?"

  □ 목적이 포함되어 있는가?
    나쁜 예 : "FTA 관세율 알려줘"
    좋은 예 : "미국 수출 시 FTA 세율 적용 여부를 판단하려면?"

  □ 구체적인 현재 상태가 포함되어 있는가?
    나쁜 예 : "파이썬 배우려면?"
    좋은 예 : "엑셀은 잘 쓰고 Python은 처음인데, 무역 데이터 정리부터 시작하려면?"

  □ 잘못된 전제는 없는가?
    체크 방법 : 질문 안에 단정적 표현("~하면 ~되는 거죠?")이 있으면 전제 교정형 사용

────────────────────────────────────────────────────────────────────────────────

🔑 핵심 포인트
- 방법1 : 빠르게 개선된 답변이 필요할 때
- 방법2 : 내 질문의 방향을 아직 확신하지 못할 때
- 방법3 : 상식처럼 알고 있지만 확신이 없는 정보를 물어볼 때
- 셀프 체크리스트로 먼저 질문을 다듬으면 방법1만으로도 충분한 경우가 많음

## 팩트체크 필요 항목

🔴 반드시 확인
  - 인증수출자 자격 요건 및 유효기간
    → 요건 미충족 시 자율 발급 불가, 추가 관세 부과 가능
    → 확인처 : 관세청 (customs.go.kr)

  - 원산지 판정 기준 (세번변경·부가가치 기준)
    → 품목별로 기준이 다르며, 잘못 적용 시 FTA 혜택 박탈
    → 확인처 : FTA 포털 (ftaportal.go.kr)

🟡 확인 권장
  - 원산지 증명서 유효기간 (통상 1년, 협정마다 상이)
    → 미국 수입자가 요구하는 시점 기준으로 재확인 필요

  - 자율 발급 서식 최신 버전 여부
    → 관세청에서 주기적으로 업데이트

🟢 참고 수준
  - 일반적인 발급 소요 기간 (기관 발급 기준 약 1~3일)
  - 발급 수수료 (대한상의 기준 약 5,000~10,000원)

💡 리플렉션 효과를 높이는 핵심 전략

  전략 1 — 페르소나 비평 추가
    비평 단계에서 특정 전문가의 관점을 부여하면 훨씬 날카롭고 구체적인 지적이 가능.
    예시 : "시니어 파이썬 개발자의 관점에서 이 코드의 효율성과 보안성을 비평해줘."
    예시 : "수출 전문 관세사의 관점에서 이 체크리스트의 법적 위험 요소를 비평해줘."

  전략 2 — 생각의 단계 명시 (Chain of Thought)
    점수를 매기기 전 판단 근거를 먼저 서술하게 하면 리플렉션의 깊이가 달라짐.
    예시 : "각 항목의 점수를 부여하기 전, 해당 점수를 준 근거를 한 줄씩 먼저 기술하세요."

⚠️ 자주 하는 실수
- 초안만 요청하고 비평·개선 단계를 생략 → 리플렉션 효과 없음
- 비평 기준을 지정하지 않음 → AI가 표면적인 비평만 수행
- 반복 횟수를 너무 많이 설정 (5회 이상) → 개선 효과 체감 감소, 2~3회가 최적
- 평가 점수만 요청하고 근거 생략 → 점수의 신뢰도 하락
