[CHAPTER_TITLE] 1장. 프롬프트 유형 & 샷 기법
[CHAPTER_DESC] 프롬프트를 어떤 방식으로 구성하느냐에 따라 AI의 응답 품질과 재사용성이 달라진다. 유형 3가지와 예시 수에 따른 샷 기법을 익히는 것이 프롬프트 작성의 기본이다.

----------------------------------------------------------------
[SECTION] 프롬프트의 3가지 유형
----------------------------------------------------------------

[SUBSECTION] 서술형 (Descriptive)
개념: 맥락과 배경을 문장으로 서술하여 AI가 상황을 이해하도록 유도하는 방식
특징: 재사용율 낮음 / 유연성 높음 / 토큰 수 많이 사용
언제 쓰나: 상황 설명이 복잡하거나, 맥락 전달이 중요한 경우

[PROMPT_BOX]
너는 [역할/경력]이야.
[대상]이 [상황]인 상황이야.
이 사람에게 [요청 내용]을 설명해줘.

[PROMPT_BOX_EXAMPLE]
너는 10년차 무역 전문가야.
중소기업 수출 담당자가 HS code를 처음 배우는 상황이야.
이 사람에게 HS code의 개념과 중요성을 설명해줘.

----------------------------------------------------------------
[SUBSECTION] 지침형 (Instructive)
개념: 단계별 절차나 명확한 규칙을 번호 목록으로 제시하여 원하는 출력을 유도하는 방식
특징: 재사용율 중간 / 유연성 중간
언제 쓰나: 출력 형식이나 작업 규칙을 엄격하게 제어해야 할 때

[PROMPT_BOX]
다음 규칙에 따라 [작업 내용]을 해줘.
1. [규칙 1]
2. [규칙 2]
3. [규칙 3]
4. [규칙 4]

[PROMPT_BOX_EXAMPLE]
다음 규칙에 따라 python 코드를 작성해.
1. 함수명은 snake_case로 작성
2. 각 함수에 docstring 포함
3. 예외 처리 반드시 포함
4. 결과는 json 파일로 만들기

----------------------------------------------------------------
[SUBSECTION] 함수형 (Functional)
개념: 입력값(변수)과 출력값(구조)을 명확히 정의하여 일관된 결과를 반복 생성하는 방식
특징: 재사용율 높음 / 유연성 낮음 / 토큰 수 절약 / 전문적
언제 쓰나: 동일한 형식의 작업을 반복할 때, 자동화 목적

[PROMPT_BOX]
입력 : {변수1}, {변수2}, {변수3}
처리 : [처리 내용]
출력 : {
  "항목1" : " ",
  "항목2" : [ ],
  "항목3" : [ ]
}

[PROMPT_BOX_EXAMPLE - 템플릿]
입력 : {product_name}, {hs_code}, {country}
처리 : 해당 국가의 관세율 및 수입 규제 조회
출력 : {
  "관세율"   : " ",
  "규제사항" : [ ],
  "필요서류" : [ ]
}

[PROMPT_BOX_EXAMPLE - 실제 적용]
입력 : {반도체}, {8501}, {대만}
처리 : 해당 국가의 관세율 및 수입 규제 조회
출력 : {
  "관세율"   : " ",
  "규제사항" : [ ],
  "필요서류" : [ ]
}

----------------------------------------------------------------
[COMPARE_TABLE] 유형별 비교
유형    | 재사용율 | 유연성 | 특징
서술형  | 낮음     | 높음   | 토큰 수 많이 사용, 맥락 전달에 강점
지침형  | 중간     | 중간   | 규칙 기반 출력 제어에 적합
함수형  | 높음     | 낮음   | 토큰 수 절약, 반복 작업 자동화에 강점
----------------------------------------------------------------

================================================================
[SECTION] 샷(Shot) 프롬프팅 기법
개념: AI에게 제공하는 예시 수에 따라 응답의 패턴 학습 정확도가 달라진다.
      예시가 많을수록 AI가 원하는 출력 형식을 더 정확히 모방한다.
================================================================

[SUBSECTION] 제로샷 (Zero-shot)
개념: 예시 없이 바로 지시하는 방식
언제 쓰나: 단순하거나 일반적인 작업 / AI가 이미 충분히 학습한 유형

[PROMPT_BOX]
[지시 내용만 바로 입력]

예: "이 제품 정말 맘에 들어" → 감정 분류 요청 (예시 없음)
예: "강아지에 대해 설명해줘" → 설명 요청 (예시 없음)

----------------------------------------------------------------
[SUBSECTION] 원샷 (One-shot)
개념: 예시를 1개 제공하는 방식
언제 쓰나: 출력 형식이 단순하거나 명확한 경우

[PROMPT_BOX]
[지시 내용]
예시) "[입력 예시]" → [출력 예시]

[처리 대상: 실제 입력값]

[PROMPT_BOX_EXAMPLE]
감정을 분류해줘.
예시) "배송이 너무 늦어 실망했어요" → 감정 : 부정

처리: "이 제품 정말 마음에 들어요!"

----------------------------------------------------------------
[SUBSECTION] 퓨샷 (Few-shot)
개념: 예시를 2개 이상 제공하는 방식
언제 쓰나: 출력 형식이 복잡하거나 일관성이 중요한 경우 / 도메인 특화 작업

[PROMPT_BOX - 감정 분류]
아래 고객의 피드백을 보고 감정(긍정/부정/중립)과 핵심 키워드를 분류해줘.

[예시 1] ← AI 학습용
입력: "배송이 너무 늦어 실망했어요."
결과: 감정 : 부정 / 키워드 : 배송 지연

[예시 2] ← AI 학습용
입력: "가격 대비 품질이 정말 훌륭합니다. 재구매 의사 있어요!"
결과: 감정 : 긍정 / 키워드 : 가성비, 품질 만족

[예시 3] ← AI 학습용
입력: "상품은 사진과 똑같네요. 그냥 무난합니다."
결과: 감정 : 중립 / 키워드 : 디자인 일치

[처리 대상] ← 실제 AI가 분류할 입력
입력: "{분류할 피드백 내용}"
결과: 감정 :      / 키워드 :

[PROMPT_BOX - HS code 분류]
HS code를 분류해줘.

[예시] ← AI 학습용
품목 : "면으로 만든 남성용 티셔츠"       → 6109.10
품목 : "폴리에스테르 소재 여성 블라우스"  → 6206.40
품목 : "아동용 데님 바지"               → 6203.42 (또는 6204.62)

[처리] ← 위 예시 패턴 적용
품목 : "{분류할 품목명}" →

[CHAPTER_TITLE] 2장. 페르소나 패턴
[CHAPTER_DESC] AI에게 특정 역할을 부여하면 같은 질문에도 전문성·어조·분석 관점이 완전히 달라진다. AI는 페르소나에 맞는 지식 체계와 언어 습관을 활성화하여 응답한다.

----------------------------------------------------------------
[SECTION] 페르소나의 효과
테스트 질문: "이 계약서 검토해줘"

[COMPARE_TABLE]
페르소나    | 응답 방향
없음        | 일반적인 설명 (특정 관점 없음)
변호사      | 법적 리스크 중심 분석 (조항별 법적 효력 검토)
CFO         | 재무조건·수익성 관련 분석 (비용·수익 구조 검토)
신입사원    | 쉽고 친절한 설명 (전문용어 최소화, 핵심만 요약)
----------------------------------------------------------------

[PROMPT_BOX - 페르소나 기본 템플릿]

구성 요소 3: 전문 분야 → 응답 범위 설정
구성 요소 4: 말투/태도 → 커뮤니케이션 스타일 설정

[CHAPTER_TITLE] 3장. 마크다운을 이용한 프롬프트 작성법
[CHAPTER_DESC] AI는 텍스트를 구조적으로 파싱(문맥상 워드를 쪼개서 저장)한다. 마크다운을 이용하면 역할·조건·출력형태가 명확히 구분되어 더 정확하고 일관된 응답을 이끌어낼 수 있다.

----------------------------------------------------------------
[SECTION] 마크다운 핵심 요소 6가지

[ELEMENT_CARD]
요소: # 헤더
역할: 프롬프트의 뼈대 / 섹션 간 경계를 명확히 구분
사용법: # → 제목 / ## → 부제목 / ### → 소제목
효과: AI가 각 섹션의 역할을 즉시 파악

[PROMPT_BOX_EXAMPLE]
# 역할
당신은 10년 경력의 관세사입니다.
# 작업
HS CODE를 분류하세요.
# 출력형식
json으로 반환

[ELEMENT_CARD]
요소: ``` 또는 """ (코드블럭)
역할: 입력 데이터 격리
사용법: 지시문과 데이터 사이에 코드블럭으로 감싸기
효과: AI가 코드블럭 안의 내용을 "처리할 데이터"로 인식, 지시문과 혼동 방지

[PROMPT_BOX_EXAMPLE]

[STRENGTH_SPECTRUM]
보안에 신경써주세요 → 취약점을 반드시 점검하고 발견 시 즉시 수정방안을 제시하세요

생각해주세요 → 고려하세요 → 반드시 우선순위에 포함하세요

좋을 것 같아요 → 권장합니다 → 예외 없이 적용하세요
번역해주세요 → 정확하게 번역하세요 → 원문의 뉘앙스까지 그대로 유지하며 번역하세요

[SECTION] 이용자 페르소나 설계 체크리스트
TIP: 프롬프트 작성 전 아래 항목을 먼저 점검하면 AI가 더 정밀하게 맞춤형 답변을 생성한다.

[CHECKLIST]
ㅁ 이 사람의 직업과 역할은?
   → AI가 응답의 전문성 수준과 관련 맥락을 설정하는 기준

ㅁ 이 주제에 대한 사전 지식은 어느 정도인가?
   → 초보 = 비유 중심 / 전문가 = 기술 용어 직접 사용

ㅁ 이 정보로 무엇을 결정하거나 실행할 예정인가?
   → 이론 중심 vs 실행 중심 중 어느 쪽으로 답할지 결정하는 기준

ㅁ 지금 어떤 상태인가? (급함 / 여유)
   → 급함 = 결론 먼저, 핵심만 / 여유 = 배경 설명 포함

ㅁ 얼마나 긴 답변을 읽을 여유가 있는가?
   → 여유 없음 = 3줄 이내 요약 / 여유 있음 = 상세 설명 허용

ㅁ 가장 두려워하는 것은? / 가장 원하는 것은?
   → 두려움 = AI가 리스크를 먼저 짚어줌
   → 원하는 것 = AI가 해당 목표를 향한 답변 구성

ㅁ 절대 쓰면 안 되는 언어나 표현이 있나?
   → 네거티브 프롬프팅으로 명시하면 더 정확하게 적용됨

[CHAPTER_TITLE] 6장. 레시피 / 뒤집힌 상호작용 / 인지 검증자
[CHAPTER_DESC] 실행 가능한 단계별 지침을 받는 레시피 패턴, AI가 먼저 질문하게 하는 뒤집힌 상호작용 패턴, 복잡한 문제를 하위 질문으로 분해하는 인지 검증자 패턴을 다룬다.

================================================================
[SECTION] 레시피 패턴
개념: 일반 질문은 두루뭉실한 방향만 제시하지만, 레시피 패턴은 목표·현재 상태·출력 형식을 명시하여 즉시 실행 가능한 순서별 행동 지침을 생성한다.
핵심 효과: "다음에 무엇을 해야 할지 고민하는 시간"을 제거 / 혼자 진행할 때 막막함을 없애주는 패턴
================================================================

[COMPARE_TABLE]
일반 질문   | "수출 자동화 어떻게 해?" | 개념 설명 + 두루뭉실한 방향 (실행 불가)
레시피 패턴 | "수출 자동화를 위한 단계별 절차를 주세요" | 순서별 행동 지침 (즉시 실행 가능)

[PROMPT_BOX - 유형 1: 기본 단계형]
# 목표
{달성하고 싶은 목표}

# 현재 상태
- {현재 보유한 것 / 상황}
- {현재 부족한 것 / 제약}

# 요청
위의 목표를 달성하기 위한 단계별 레시피를 작성하세요.
각 단계는 반드시 아래 형식을 그대로 사용하세요.

## STEP {번호} : {단계명}
- 할 일      : {지금 당장 해야 할 구체적 행동}
- 소요시간   : {예상 기간 또는 시간}
- 완료 기준  : {이것이 되면 다음 단계로 넘어갈 수 있음}
- 주의사항   : {초보자가 자주 하는 실수}

[PROMPT_BOX_EXAMPLE]
# 목표
python 초보자가 무역 HS code 자동 분류 프로그램 만들기

# 현재 상태
- python 설치만 되어 있음
- 프로그램 개발 경험 없음
- 엑셀로 품목 데이터 500개 보유

# 요청
위의 목표를 달성하기 위한 단계별 레시피를 작성하세요.
각 단계는 반드시 아래 형식을 그대로 사용하세요.

## STEP {번호} : {단계명}
- 할 일      : {지금 당장 해야 할 구체적 행동}
- 소요시간   : {예상 기간 또는 시간}
- 완료 기준  : {이것이 되면 다음 단계로 넘어갈 수 있음}
- 주의사항   : {초보자가 자주 하는 실수}

[PROMPT_BOX - 유형 2: 분기 포함형]
{절차 설명 요청}.
단, 각 단계에서 발생할 수 있는 분기 상황도 포함하세요.

형식:
STEP N. {행동}
- 정상 : {이 결과가 나오면 → 다음 단계로 진행}
- 이상 : {이 결과가 나오면 → 어떻게 대응해야 하는지}

[PROMPT_BOX - 유형 3: 체크리스트형]
{작업명}을 위한 실행 체크리스트를 작성해줘.

형식:
[  ] {실행해야 할 행동} - {왜 해야 하는지 또는 주의사항}

[PROMPT_BOX - 유형 4: 병렬 레시피형]
{목표}를 달성하기 위한 {N}일간의 병렬 실행 계획을 작성하세요.
각 날짜마다 동시에 진행 가능한 작업들을 병렬로 배치하세요.

형식:
| 1일차     | 2일차     | 3일차     |
|-----------|-----------|-----------|
| ㅁ 작업 A | ㅁ 작업 C | ㅁ 작업 E |
| ㅁ 작업 B | ㅁ 작업 D | ㅁ 작업 F |

[SECTION] 뒤집힌 상호작용 패턴
개념: 일반 패턴은 사용자가 정보를 주면 AI가 즉시 결과물을 생성하지만, 이 패턴은 AI가 먼저 필요한 정보를 질문하여 정보가 충분히 수집된 후에만 결과물을 생성한다.
핵심 효과: 정보 부족으로 인한 부정확한 결과물 생성 방지
================================================================

[PROMPT_BOX - 방법 1: 일괄 질문형]
제가 원하는 결과물은 {목표}입니다.
바로 만들지 말고, 먼저 아래 항목들을 질문해서
제 답변을 모두 수집한 후 작성하세요.

수집이 필요한 정보:
- {수집 항목 1}
- {수집 항목 2}
- {수집 항목 3}

[PROMPT_BOX - 방법 2: 순차 질문형]
제가 원하는 결과물은 {목표}입니다.
한 번에 하나의 질문만 하세요.
제가 답하면 다음 질문으로 넘어가고,
충분한 정보가 모이면 결과물을 작성하세요.

※ 질문은 5개의 선택지를 제시하고 번호로 선택 가능하게 해주세요.
※ {N}~{M}개의 질문이 끝나면 맞춤형 {결과물}을 작성해주세요.

[COMPARE_TABLE - 언제 어느 방법을 쓸까]
항목             | 방법 1 (일괄 질문)           | 방법 2 (순차 질문)
질문 방식        | 한 번에 모든 질문 제시        | 한 번에 하나씩 순서대로
적합한 상황      | 수집 항목이 미리 정해진 경우  | 항목이 유동적이거나 대화형
사용자 부담      | 여러 항목 한꺼번에 답해야 함  | 하나씩 답하면 되어 부담 적음
결과물 생성 시점 | 모든 답변 수집 후 1회 생성   | 마지막 질문 완료 후 1회 생성

================================================================
[SECTION] 인지 검증자 패턴
개념: 복잡한 질문을 바로 입력하면 AI가 즉각 답변하면서 오류·누락 가능성이 높아진다. 이 패턴은 AI가 먼저 문제를 하위 질문으로 분해하고 각 질문을 순서대로 검증한 뒤 통합된 판단을 내리도록 유도한다.
핵심 효과: 복잡한 판단일수록 성급한 결론 대신 단계적 검증으로 정확도를 높임
핵심 문구: "바로 답하지 말고" ← 이 한 문장이 AI의 즉각 응답을 억제하고 단계적 검증 모드로 전환
================================================================

[COMPARE_TABLE]
일반 패턴       | 복잡한 질문 입력 → 즉각 답변 → 오류·누락 가능성 높음
인지 검증자 패턴 | 복잡한 질문 입력 → 하위 질문 분해 → 순서대로 검증 → 통합 판단 → 정확도 향상

[PROMPT_BOX - 방법 1: 명시적 분해 요청]
아래 질문에 바로 답하지 말고
먼저 이 질문을 정확히 답하기 위해 필요한 하위 질문을 {N}~{M}개 도출하세요.
각 하위 질문에 순서대로 답한 뒤
최종 통합 답변을 제시하세요.

질문: {복잡한 질문 입력}

[PROMPT_BOX - 방법 2: 전문가 검증 방식]
{전문 분야} 전문가처럼 이 문제를 다루세요.
확신이 없는 부분은 "가정"으로 명시하고,
각 판단의 근거를 함께 제시하면서 단계적으로 결론에 도달하세요.

질문: {복잡한 질문 입력}

[PROMPT_BOX_EXAMPLE]
아래 질문에 바로 답하지 말고
먼저 이 질문을 정확히 답하기 위해 필요한 하위 질문을 3~5개 도출하세요.
각 하위 질문에 순서대로 답한 뒤
최종 통합 답변을 제시하세요.

질문: python으로 무역 데이터 자동화를 구축할 때 클라우드 서버가 필요한가요?

[COMPARE_TABLE - 패턴별 적합한 상황]
명시적 분해 요청  | 질문이 여러 도메인에 걸쳐 있고 변수가 많을 때
전문가 검증 방식  | 특정 전문 지식 기반의 단계적 논증이 필요할 때
하위 질문 수 지정 | AI의 분해 깊이를 통제하고 싶을 때 (3~5개 등 명시)
